Challenge: Forgotten Notes
Category: Web
Difficulty: Easy
Author: SYPHERR
Description
Aether Security claims to have the most secure corporate infrastructure. They recently migrated their public portal to containerized systems and had an external company review their external perimeter. However, we suspect one of their developers left behind some configuration files and temporary internal developer logs. Can you investigate the site, trace the clues left by the engineering team, and retrieve the flag?
Flag Format: BSCTF{...}
Given URL
http://<challenge-ip>:<port>
______________
Solution
Step 1: Homepage Source Code Audit
Load the homepage (index.html) and view the HTML page source (right-click -> View Page Source). Scroll to the bottom of the document to discover a hidden developer HTML comment:
<!-- TODO(deploy-team): Clean up staging assets. The migration guidelines and development notes are still accessible at /dev-notes-9a2f.html. Please archive this before the next audit. -->
______________
Step 2: Inspect Staging Dev Notes
Navigate directly to the unlisted path discovered in the comment: /dev-notes-9a2f.html. Read through the internal wiki. Under the "Temporary Staging Environment Console" card, locate the active diagnostics console panel endpoint:
/staging-console-8c3b.html
______________
Step 3: Retrieve and Decode Flag
Navigate to /staging-console-8c3b.html to view the diagnostics panel. Under the "Active Process Environment Variables" table, locate the SYS_FLAG key containing the Base64-encoded string:
QlNDVEZ7YWx3YXlzX2ZvbGxvd190aGVfY2x1ZXN9

Convert the Base64 string back into plaintext to obtain the final flag.
______________
Flag
BSCTF{always_follow_the_clues}
